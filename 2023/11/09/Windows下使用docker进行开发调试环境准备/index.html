Windows下使用docker进行开发调试环境准备

发布于: 2023-11-09

现在的软件项目开发调试日趋复杂，经常需要一堆的软件环境才能正常的进行开发调试。比如数据库、redis、mq等等，这些软件安装麻烦还不好管理，平时不用还占着资源。有没有一种办法解决这个问题呢？答案是有的，我们可以借助docker来完成软件环境的准备，本文将在windows10/11的环境下演示使用docker进行开发调试环境准备。

  • WSL2环境准备

    控制面板->程序->启用或关闭Windows功能->向下滑 勾选以下两项功能。

  • 安装启动postresql数据库

    本节内容将完成postgresql的镜像启动，并将涉及docker的一个重要概念：数据卷的运用
    数据卷的定义：Docker数据卷是一个可以在Docker容器和主机之间共享数据的特殊目录。它允许容器在不影响主机文件系统的情况下访问主机上的文件，并且可以在多个容器之间共享数据。使用数据卷可以方便地将数据持久化保存，并且在容器删除后仍然保留数据。数据卷是Docker中非常方便的功能，可以帮助用户更好地管理和维护容器化应用程序。
    简略版：你如果不想数据随着容器的删除而消失，就必须独立于容器之外的数据卷
    综上所述：对于启用postgresql容器，使用数据卷挂载是非常必要的

命令详解

命令名称 解释及描述
–name 容器名称
–restart always 容器随docker启动而自动启动
-e TZ=’Asia/Shanghai’ 设置时区
-e POSTGRES_PASSWORD=’1234qwer 设置密码为1234qwer
-e ALLOW_IP_RANGE=0.0.0.0/0 允许远程访问
-v pgdata:/var/lib/postgresql/data -p 55435:5432 -d postgres 挂载数据目录到卷：pgdata（也就是我们第一步创建的数据卷）
-p 55435:5432 设置端口映射。55435（宿主机端口）:5432（容器端口）

其他常见的docker run命令

镜像 命令
ldap docker run –restart always -p 389:389 -p 636:636 –name my_ldap –network bridge –hostname openldap-host –env LDAP_ORGANISATION=”passport” –env LDAP_DOMAIN=”passport.com” –env LDAP_ADMIN_PASSWORD=”1234qwer” -v ldapdata:/etc/ldap/slapd.d -v ldapconfig:/var/lib/ldap –detach osixia/openldap
rabbitmq docker run -d –hostname my-rabbit –name rabbit –restart always -e RABBITMQ_DEFAULT_USER=user -e RABBITMQ_DEFAULT_PASS=password -p 15672:15672 -p 5672:5672 rabbitmq:management
redis docker run –restart=always -p 6379:6379 –name redis -d redis
zookeeper docker run -d -e TZ=”Asia/Shanghai” -p 2181:2181 -v $PWD/data:/data –name zookeeper –restart always zookeeper
postgres docker run -it –name postgress –restart always -e TZ=’Asia/Shanghai’ -e POSTGRES_PASSWORD=’1234qwer’ -e ALLOW_IP_RANGE=0.0.0.0/0 -v pgdata:/var/lib/postgresql/data -p 55435:5432 -d postgres
nginx docker run –restart always -d -p 80:80 –name my-nginx -v nginx-config:/etc/nginx/conf.d nginx
